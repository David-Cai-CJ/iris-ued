class iris.gui.pyqtgraph.util.garbage_collector.GarbageCollector(interval=1.0, debug=False)[source]¶

Bases: object

Disable automatic garbage collection and instead collect manually on a timer.

This is done to ensure that garbage collection only happens in the GUI thread, as otherwise Qt can crash.

Credit: Erik Janssens Source: http://pydev.blogspot.com/2014/03/should-python-garbage-collector-be.html

iris.gui.pyqtgraph.util.lru_cache module¶

class iris.gui.pyqtgraph.util.lru_cache.LRUCache(maxSize=100, resizeTo=70)[source]¶

Bases: object

This LRU cache should be reasonable for short collections (until around 100 items), as it does a sort on the items if the collection would become too big (so, it is very fast for getting and setting but when its size would become higher than the max size it does one sort based on the internal time to decide which items should be removed – which should be Ok if the resizeTo isn’t too close to the maxSize so that it becomes an operation that doesn’t happen all the time).

class iris.gui.pyqtgraph.util.mutex.Mutex(*args, **kargs)[source]¶

Bases: PyQt5.QtCore.QMutex

Subclass of QMutex that provides useful debugging information during deadlocks–tracebacks are printed for both the code location that is attempting to lock the mutex as well as the location that has already acquired the lock.

Also provides __enter__ and __exit__ methods for use in “with” statements.
